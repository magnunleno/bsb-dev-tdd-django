TDD - Test Driven Development

  • Não vou me aprofundar nos conceitos;
  • Processo de desenvolvimento de software;
  • Kent Beck popularizou o conceito no XP em 1999;
  • Lifecycle simplificado
    1. Transforme as especificações em casos de tests;
    2. Implemente somente a funcionalidade para validar o caso de testes;
    3. Refatore (se necessário)
    4. GOTO 1

Etapas

  • Planeje
    • Novas funcionalidades...
    • Modificações de funcionalidades existentes...
    • API (não é a API REST)...
  • Escreva os Testes
    • Unitários e Funcionais
  • Escreva o Código
    • Até que todos os testes passem!
  • REFATORE!

Testando, 1... 2... 3...

  • Não faça testes manuais;
  • Utilize dados fictícios;
  • ...ou extraídos de produção (descaracterizados);
  • Valide toda sua lógica (cobertura);
  • Se está difícil de testar, você não está fazendo certo!
  • Utilize mocking, stubs e spies;

Spies, Stubs e Mocks

  • De acordo com Martin Fowler...
  • São sub-tipos de test doubles, junto com Dummy e Fake Objects;
  • Nomenclatura varia de acordo com a linguagem e o framework de testes;

Spies

  • Um "envelope" de uma função ou método sem alterar seu comportamento;
  • Registra seus argumentos, quantidade de vezes que foi invocada, retorno...

# utils.py
import datetime


def fmt_datetime(dt):
    if type(dt) == date:
        dt = datetime.datetime(
          dt.year,
          dt.month,
          dt.day,
        )
    return dt.isoformat()


# test_utils.py
from unittest.mock import patch
from datetime import date, datetime

import utils


def test_fmt_datetime():
    with patch.object(utils, 'datetime', wraps=utils.datetime) as spie:
        fmt = utils.fmt_datetime(
          date(2018, 1, 1)
        )
        assert spie.called
        assert spie.call_args[0] == (2018, 1, 1)
        assert len(spie.mock_calls) == 1
        assert fmt == "2018-01-01T00:00:00"

        spie.reset_mock()

        fmt = utils.fmt_datetime(
          datetime(2018, 1, 1, 2, 3, 4)
        )
        assert not spie.called
        assert fmt == "2018-01-01T02:03:04"

Stubs

  • Substituição de um método ou função simples a fim de evitar uma "comunicação externa";
  • Geralmente envolve a devolução de "respostas enlatadas";

Stubs


# utils.py
import os

def apaga_arquivo(caminho):
    if os.path.exists(caminho):
        os.remove(caminho)
        return True
    return False


# test_utils.py
import utils
from unittest.mock import patch

@patch('utils.os.remove', return_value=True)
def test_apaga_arquivo(mock):
    with patch('utils.os.path.exists', return_value=True):
        success = utils.apaga_arquivo('/phony/file')
        assert success

    with patch('utils.os.path.exists', return_value=False):
        success = utils.apaga_arquivo('/phony/file')
        assert not success

Mock

  • Stub de um "objeto complexo";
  • Possui funcionalidades de spies e de stubs

Mock


class Cliente:
    def __init__(self, id=None, nome=None):
        self.id = id
        self.nome = nome

    @classmethod
    def use(kls, api=None):
        if api:
            kls.api = api

    @classmethod
    def por_id(kls, id):
        data = kls.api.fetch(id)
        if data is None:
            return None
        return kls(**data)


from unittest.mock import MagicMock

def test_cliente_api():
    mock = MagicMock()
    utils.Cliente.use(api=mock)

    mock.fetch.return_value = {"id": 1, "nome": "Fulano"}
    c = utils.Cliente.por_id(1)
    assert mock.fetch.call_count == 1
    assert mock.fetch.call_args[0] == (1,)
    assert c.id == 1
    assert c.nome == "Fulano"

    mock.fetch.reset_mock()
    mock.fetch.return_value = None
    c = utils.Cliente.por_id(3)
    assert c is None
    assert mock.fetch.call_count == 1
    assert mock.fetch.call_args[0] == (3,)
